# A股动态参数量化交易系统 - 项目开发文档 (Project Prompt)

> **修订版 v2.3** — 2026-06-25  
> 数据管线：300只股250交易日数据拉取中  
> **v2.3 重构**：9策略(趋势+反脆弱+行为金融) + 权重回归 + 随机搜索替代 Optuna

---

## 1. 项目概述

### 1.1 核心理念

1. **不做价格预测也不选策略** — LSTM 输出 9 个策略的资金分配权重，让市场状态决定哪个策略主导
2. **权重回归** — 多策略共振=高置信=大资金，单策略独走=低置信=小资金
3. **分钟级粒度** — 240条(1天)特征 → 1200条预热+1200条回测(5交易日)
4. **三类策略互补** — 趋势(顺涨)、反脆弱(抄恐慌)、行为金融(抓拐点)同时工作

### 1.2 技术架构

```
数据采集(zzshare→TDengine) → 特征工程(32维:个股+大盘+截面排名) → 样本生成(随机搜索→权重标签)
→ 模型训练(LSTM→9维softmax权重) → FastAPI推理 → SpringBoot → Vue
```

### 1.3 v2.3 关键变化

| 旧 v2.2 | 新 v2.3 |
|---------|---------|
| 5 个趋势策略，Optuna 选最优 | **9 策略(5趋势+2反脆弱+2行为金融)，随机搜索** |
| LSTM 输出策略 ID (分类) | **LSTM 输出 9 维权重 (回归)** |
| MACD/KDJ 硬门槛过滤 | **去掉，让数据说话** |
| Optuna 串行搜索 135 trials | **并行随机采样 100 trials，快 10 倍** |
| 标签 = 最优策略 | **标签 = softmax(Calmar/0.5) 权重向量** |

## 4. 样本生成

### 4.1 窗口设计

```
|-- 特征窗口(240条=1天) --|-- 预热区(1200条=5天) --|-- 回测区(1200条=5交易日) --|
      ↑ LSTM 输入             ↑ 指标计算(不交易)        ↑ 实际模拟
```

步长=240(1天), 每只股约 240 个窗口。

### 4.2 多批次资金池

```
单资金池 50,000
├── 现金: 动态变化
└── 持仓批次 list[Lot], FIFO, ≤10个, 各批次独立T+1

LSTM输出的9个权重作用:
  权重 × 策略信号 = 每股买入信号强度
  信号强度 × 可用现金 × buy_ratio = 实际买入金额
  多策略共振 → 信号强度高 → 仓位重
  单策略独走 → 信号强度低 → 仓位轻
```

### 4.3 随机搜索标注 (替代 Optuna)

```
每窗口:
  1. 随机采样 100 组 (策略ID, p1-p5, 通用参数6个)
  2. 10 线程并行跑 100 次回测
  3. 按策略ID分组, 取每组最高 Calmar: C₀..C₈
  4. 权重标签 = softmax([C₀..C₈] × 2)   (T=0.5 即放大2倍)

  Plan B:
    正样本: weight vector + 每组最优参数
    负样本×3: 扰动权重 + 每组最差参数
```

耗时: 100 trials × 8ms(单次回测平均) / 10 线程 ≈ 80ms/窗口。比 Optuna 快 6 倍。

### 4.4 LSTM 输出设计 (后续)

```
输入: (240, 32)
  │
LSTM(128×2) + MLP
  │
  └── 权重头 → Linear(128→64)→ReLU→Linear(64→9)→Softmax
       Loss: KLDivLoss(权重标签, 预测权重)
```

## 6. 关键注意事项

1. 三个策略类别不会冲突——各自在特定市场状态下触发
2. 权重标签用 softmax(Calmar/0.5) 成生——Calmar 差异被放大 2 倍
3. 随机搜索 100 trials 全部独立，天然并行，无串行依赖
4. 多批次持仓上限 10 个，FIFO 卖出，窗口末不强制平仓
5. 涨跌停按板块逐日更新，ATR(480) ≈ 2天真实波幅
6. 印花税单向(卖出千1)，佣金万1，滑点 0.05%
