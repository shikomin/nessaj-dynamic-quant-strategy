## 二、Java 后端 — K线查询 API

### `GET /api/market/kline`
- 参数: `code`, `period`(1m/5m/15m), `start`, `end`, `limit`(默认240)
- 无时间: `ORDER BY ts DESC LIMIT 240` 取最新 240 条 → Java 内存反转
- 5m/15m: TDengine `INTERVAL(5m)` / `INTERVAL(15m)` 降采样
  - `FIRST(open), MAX(high), MIN(low), LAST(close), SUM(volume), SUM(amount)`
- `@DS("tdengine")` 切动态数据源, 原生 JDBC `ResultSet` 映射

### `GET /api/market/kline/dates?code=xxx`
- 查询 `hk_{code}` 全部 `ts` → Java 端提取唯一日期 → 倒序返回
- 供前端日历日期选择器使用

## 三、Canvas K线引擎 (KlineEngine.ts)

### 架构
- 纯 TypeScript Canvas 渲染, 无第三方依赖
- DPR 感知: `canvas.width = w*dpr`, `ctx.setTransform(dpr,0,0,dpr,0,0)` 高清
- 渲染层级: 网格→边框→成交量→蜡烛→时间轴→价格轴→十字光标→蜡烛弹框

### 分段拼接时间轴 (setDay + tsToX)
- `setDay(dateStr)`: 构造固定两段 `9:30-11:30` / `13:00-15:00`
- `tsToX`: 只累加视口内有效交易时长, lunch gap 塌陷不占像素
- 无 segments 时自动降级为线性映射 (引擎创建到 setDay 之间)
- **Bug**: 动态 gap 检测 (buildSegments) 对 5m/15m 周期失效 → 每根变独立段 → 全挤中间
  - **Fix**: 改为固定两段, 去掉了 buildSegments

### 缩放与平移
- `setMinPeriod(p)`: 设置缩放下限 = 最小周期 × 16
  - 1m → 16min; 5m → 80min; 15m → 4h
  - 后缀 `s`/`m` 自动选乘法因子, 支持秒级周期
- `pan`: 用 `_visibleTradingMs()`(当前视口活跃交易时间) 算像素/时间比
  - **Bug**: 此前用 `totalSegmentMs`(全天 14400s), 缩放到 5min 时换算偏差 48 倍
  - **Fix**: 改用当前视口实际覆盖的有效时间
- `clampViewport`: 视口锁在段首尾, 最小范围 periodMs×16
- 无 segments 时 zoom/pan 降级为线性模式

### 时间轴
- 字体 20px (之前 10px 看不清 `13:00` 糊成 `00:00`)
- 三标签 `09:30` / `11:30/13:00`(断点) / `15:00`
- `inView` 裁剪: 超出图表 ±50px 不画
- 断点与末尾距离 < 8px 跳过防重叠
- 视口只覆盖单段时降级为等间隔蜡烛时间标签

### 十字光标 + 蜡烛弹框
- 十字光标: 虚线竖+横, 底部时间标签 + 左侧价格标签
- `getCrosshairData()`: emit 给 Vue 右侧面板显示 OHLCV
- 蜡烛点击弹框: 高/低/开/收 四行, 自适应位置 (右/左/上/下边界保护)
