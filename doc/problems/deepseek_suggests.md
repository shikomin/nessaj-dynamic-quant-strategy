# v2.3 方案评估与修订建议

> **日期**: 2026-06-25
> **来源**: 对 v2.3 project_prompt.md 逐项审查
> **范围**: 策略矩阵 / 标签构造 / 特征工程 / 搜索架构 / 风控完整性

---

## 一、v2.3 做对了的改进

### 1.1 从分类转回归 — 建模方向修正

v2.2 LSTM 输出策略 ID（分类）隐含假设：任一时刻只有一种最优策略。现实是同一市场状态下多种策略可以共振——趋势策略打 6 分、V 型反转策略打 4 分、布林回归打 0 分。**权重分配才是真实的市场状态表征**，回归天然比分类更接近真相。

### 1.2 三类策略互补 — 覆盖面质变

```
趋势(0-4)   → 顺市赚钱
反脆弱(5-6) → 恐慌中接飞刀
行为金融(7-8) → 拐点处抄底
```

A 股最大的特点是**极端情绪驱动**（恐慌踩踏 + 贪婪追涨）。纯趋势类在恐慌中会被反复止损，反脆弱类专门吃极性场景，形成自然对冲。比 v2.2 的 5 个同质化趋势策略合理得多。

### 1.3 随机搜索替代 Optuna — 工程上聪明

Optuna 的 Bayesian 优化在单窗口只有 ~100 个数据点（参数维度 11 个）时，搜索效率被采样开销拖累。100 次随机采样 × 10 线程并行 ≈ 80ms/窗口，确实比串行 Optuna 快一个数量级。

### 1.4 截面排名特征 — 被低估的亮点

`rank_ma5`, `rank_rsi_14`, `rank_volume` 三个截面排名特征是实质性的信息增量。单纯看"我的 RSI 是 65"没有意义；"我在全市场 300 只股里 RSI 排前 5%"才有意义——这抓住了 A 股涨跌的本质是相对强弱轮动。

## 二、发现的问题

### 2.1 权重标签噪声 — 最严重问题

**当前逻辑**：

```
每窗口 100 次随机采样 → 按策略分组取 max Calmar → softmax(Calmar/0.5)
```

**问题**：100 次采样平均分给 9 个策略，每个策略只有 ~11 次随机参数探索。策略 0（MA突破）可能碰巧被采到了参数黄金组合 `fast=3, slow=45`，而策略 3（ATR通道）全采到了边缘参数组合。结果 softmax 把 60% 权重给策略 0——**标签不是"策略0更适合当前市况"，而是"策略0的参数采样更走运"**。

这会形成系统性 label noise：模型学到的是参数采样的运气分布，而非市场状态的策略适配性。训练 Loss 可能很低，但推理时泛化差——因为真实市况下不存在这种"运气偏差"。

**影响等级：P0**

### 2.2 反脆弱策略的 A 股适配性存疑

| 策略 | 信号 | 在 A 股的实际情况 |
|------|------|-------------------|
| 5 跌停次日跳空买入 | 缩量 70% + 次日低开 | 跌停股往往有基本面恶化（业绩雷、监管问询、st 风险），次日低开可能是继续跌停的前奏。T+1 买进去，第三天才能卖，隔夜风险巨大 |
| 6 缩量阴跌末端 | 连跌 3 天 + 每天量缩 20% | A 股缩量阴跌是最危险的形态——往往是主力出完货后的自由落体，非底部信号。真正底部是**放量恐慌**，不是缩量阴跌 |

这两个策略在逻辑上试图做"反脆弱"——从恐慌中获取正期望。但 A 股的恐慌更多是**基本面驱动的持续恶化**而非纯情绪超调。接飞刀的概率可能高于捡黄金。需要在回测中严格验证这两个策略的历史胜率，如果 < 30% 应考虑替换或重构。

**影响等级：P1（取决于回测验证结果）**

### 2.3 去掉确认过滤导致信号噪音放大

v2.2 的量能确认 + 趋势过滤 + 涨停距过滤本质上在做**信号提纯**：

- 无量假突破的 MA 金叉 → 过滤掉
- 熊市中 MA 金叉 → 趋势过滤拦截
- 涨停板附近追高 → 涨停距过滤拦截

v2.3 去掉全部过滤后：
- 无量假突破触发策略 0，回测结果差 → 权重标签低 → 但也消耗了搜索预算
- 牛市和熊市的策略触发量相同 → 标签区分度下降
- 模型需要从 32 维特征中"悟出"趋势过滤的存在——这是让模型做回测引擎本应做的工作

**"让数据说话"的理念是对的，但应在回测层面说话（搜出真实绩效），而不是把信号提纯的责任丢给 LSTM。** LSTM 的特征是 240×32 的矩阵，它不具备回测引擎对交易规则（涨停买不到、无量假突破）的精确建模能力。

**影响等级：P1**

### 2.4 无空仓机制 — 永远 fully invested

当前设计 LSTM 输出 9 维 softmax，权重和为 1。含义是：任何时候钱都必须在 9 个策略中分配完毕。

问题场景：
- 2015 年股灾、2018 年单边熊市、2024 年 1 月急跌 → 模型无法空仓
- 权重输出会从"趋势策略权重高"退化为"各策略权重平均"——但钱还是套在市场里
- 在牛短熊长的 A 股，空仓择时是超额收益的最大单一来源

**影响等级：P0**

### 2.5 随机搜索的分层不均衡

100 trials 全随机采样（策略 ID + 参数一起采），导致每个策略被采样次数不均。如果策略 5 只被采了 3 次，其结果严重受样本量不足影响。

实际可得情况：
- 如果 100 trials 服从均匀分布，每个策略期望 11.1 次采样
- 但策略 7、8（行为金融）可能在大部分窗口不产生交易 → Calmar = 0 → 权重被压制
- 这不是策略本身不好，而是搜索深度不足

**影响等级：P2**

## 三、修订建议

### 3.1 权重标签改为 rank-based

**当前**：
```python
weights = softmax([C₀, C₁, ..., C₈] / 0.5)
```

**建议**：
```python
# 对所有 100 次 trial 的 Calmar 排序
# 每个策略取在此窗口内的排名分位数
for i in range(9):
    rank_i = percentile_rank(max(C_i_trials), all_100_calmars)  # 0.0~1.0
weights = softmax([rank_0, rank_1, ..., rank_8] * 3)  # τ=0.33, 放大排名差异
```

**理由**：
- 排名的信噪比远高于绝对值。Calmar 可能因为随机采样涨跌停线计算偏差而整体偏移，但排序相对稳定
- 一个策略排前 20%、另一个排前 10%，这个**相对**信息比 Calmar=1.8 vs 1.2 可靠得多
- 极端情况下所有策略 Calmar < 0，rank-based 仍然有效（排名仍有高低），而 Calmar softmax 会把负值也映射成正权重

### 3.2 增加第 10 维：空仓权重

**设计**：
```python
# LSTM 输出 10 维 softmax
# 维度 0-8: 9 策略权重
# 维度 9: 空仓权重（对应现金占比）
```

**标签构造**：
```python
# 空仓的 Calmar = 0（不赚不亏）
# 将其与 9 个策略的 Calmar 一起参与 softmax
all_calmars = [C₀, ..., C₈, 0.0]  # 第 10 个是空仓
weights = softmax(rank_of(all_calmars) * 3)
```

**效果**：
- 所有策略 Calmar < 0 → 空仓权重自动变高
- 只有少数策略赚钱 → 空仓 + 好策略瓜分权重
- 牛市全面赚钱 → 空仓权重自动趋近 0
- 不需要额外引入"择时模块"，空仓能力内生在后验 Calmar 中

**改动量**：策略矩阵 +1 行（空仓），LSTM 输出 9→10 维，回测引擎不需改动（空仓 = 不开仓）。

### 3.3 随机搜索改为分层采样

**当前**：
```python
trials = random.sample(strategy_id * param_space, 100)
```

**建议**：
```python
trials = []
for strategy_id in range(9):
    # 每个策略保证 11 次采样
    trials += random.sample(strategy_id, param_space, 11)
trials += random.sample(all_strategies, 1)  # 凑 100
```

**理由**：
- 避免采样不均衡导致的标签偏差
- 每个策略都有公平的参数探索机会
- 增加 1 次额外采样可验证均衡性是否影响结果

### 3.4 保留部分关键确认过滤

不是全量恢复 v2.2 的所有过滤，而是在回测引擎中保留 3 个硬性约束（不需要 Optuna/随机搜索，直接硬编码）：

| 过滤 | 硬编码规则 | 理由 |
|------|-----------|------|
| 涨停不可买 | `close >= limit_up * 0.995` → 跳过买入 | 物理约束，非策略判断 |
| 一字板检测 | `open == high == low == close` 且 `at/above limit` → 跳过 | 无量涨停买不到，回测不应假设能买到 |
| 无量防假 | `volume < volume_ma20 * 0.3` → 跳过信号 | 极度缩量时信号大概率是噪音 |

这三个是物理/市场规则，不是策略偏好。去掉它们会让回测高估策略收益，导致训练标签系统性偏差。

### 3.5 反脆弱策略增加基本面安全网

策略 5/6 触发买入时增加硬性前置条件：

```python
# 辅助数据：从 zzshare 获取股票简称和财报日期
if stock_name_contains("ST"):
    skip  # ST 股不适用反脆弱策略

if within_3_days_of_earnings_report(stock_code):
    skip  # 财报窗口期不接飞刀

if prev_day_turnover_rate < 0.01:  # 日换手 < 1%
    skip  # 无流动性，接了也跑不掉
```

这些条件不需要 Optuna 搜索，是风控硬规则。在不显著增加搜索复杂度的前提下，拔掉最危险的场景。

## 五、修订后的标签构造伪代码

```python
def build_weight_label(calmar_by_strategy: dict[int, list[float]], 
                       all_calmars: list[float]) -> np.ndarray:
    """
    构造 10 维权重标签（9 策略 + 1 空仓）。
    
    参数
    ----
    calmar_by_strategy : {strategy_id: [trial Calmar list]}
    all_calmars        : 全部 100 trial 的 Calmar 列表
    
    返回
    ----
    weights : shape (10,) float32, sum=1
    """
    # 每策略取最优 Calmar
    best_calmars = []
    for sid in range(9):
        trials = calmar_by_strategy.get(sid, [])
        best_calmars.append(max(trials) if trials else 0.0)
    
    # 空仓 Calmar = 0
    best_calmars.append(0.0)  # 第 10 维
    
    # rank 归一化
    all_sorted = sorted(all_calmars, reverse=True)
    ranks = []
    for c in best_calmars:
        # 在全量 trial 中的百分位排名
        rank = sum(1 for x in all_sorted if x <= c) / len(all_sorted)
        ranks.append(rank)
    
    # softmax with temperature
    tau = 0.33  # 放大排名差异
    ranks = np.array(ranks, dtype=np.float32)
    logits = ranks / tau
    weights = np.exp(logits - logits.max()) / np.exp(logits - logits.max()).sum()
    
    return weights.astype(np.float32)
```
