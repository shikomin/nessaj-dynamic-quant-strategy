# 指数分钟线数据获取问题

> **日期**: 2026-06-26
> **影响**: 大盘特征 (index_change_pct, relative_strength, index_volume_ratio) 不可用
> **当前状态**: 待解决

---

## 问题描述

zzshare 的 `stk_mins()` 不支持指数分钟线 K 线拉取。

### 已测试的代码格式

| 代码 | 结果 |
|------|------|
| `000001.SH` | 返回平安银行 (000001) 的个股分钟线，close≈10.99 |
| `000001.SSE` | 同样返回个股分钟线 |
| `999999.SH` | 空 |
| `1A0001.SH` | 空 |
| `sh000001` | 空 |

### 原因

`stk_mins` 的代码解析逻辑只看数字部分查个股，不区分 `.SH`/`.SZ` 后缀是否对应指数。

### zzshare API 线索

- `daily()` 方法：可能支持 `000001.SH` 获取上证综指日线。README 文档提到支持 tushare 兼容规范（000001.SH = 上证指数）
- `plate_kline(b_code="883957")`：同花顺全 A 指数日线（代表全市场）
- `plates_rank/plates_trend`：板块级别的趋势数据
- 没有专门的 `index_mins()` 方法

当前 `i_sh000001`、`i_sz399001`、`i_sz399006` 三张指数子表中的数据全为平安银行等个股的分钟线（价格在 10-15 元区间，而非指数点位），不可用于计算大盘特征。

## 可能的解决方案

### 方案 A：`daily()` 获取指数日线 → 广播到分钟

```python
# 用 daily() 获取上证指数日线
df = api.daily(ts_code='000001.SH', start_date='20250601', end_date='20260622')
# 把日线涨跌幅广播到当天的所有分钟线
```

优势：改动小，zzshare 原生支持
劣势：大盘特征从分钟级降为日级，信息量降低

### 方案 B：自建 300 股迷你指数

用已有 300 只股票的分钟线自己算大盘指标：
- `index_change_pct` = 300 只股该分钟的涨跌幅中位数
- `relative_strength` = 个股涨跌幅 - 300 股涨跌幅中位数
- `index_volume_ratio` = 300 股中位量比

优势：零外部依赖，更贴近实际交易池
劣势：非真实大盘

### 方案 C：放弃大盘特征

退回到 29 维纯个股特征。
